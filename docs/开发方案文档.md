# Tribo - 活动跟踪系统 开发方案文档

> 版本: v1.0 | 平台: 微信小程序 | 技术栈: Python FastAPI + MySQL/PostgreSQL + 微信云开发

---

## 一、项目概述

### 1.1 项目名称
**Tribo** — 取自 "Tribe"（部落）+ "Tracking"（跟踪），寓意把一群人/个人的活动凝聚在一起。

### 1.2 核心理念
用户可以通过 Tribo 跟踪四大类活动：
1. 🏋️ **健身运动** — 打卡、记录训练数据
2. 👥 **团队活动** — 发布、报名、签到
3. ✅ **个人习惯/任务** — To-do、复盘
4. 🏃 **专项活动**（跑步/骑行/读书等）— 分类跟踪

### 1.3 目标用户
- 个人用户：日常运动、习惯管理
- 小团队/社群：共同活动组织
- 健身爱好者/跑团/书友会

---

## 二、技术选型

### 2.1 微信小程序端
| 技术 | 说明 |
|------|------|
| WXML + WXSS | 原生小程序框架 |
| JavaScript / TypeScript | 逻辑层 |
| WeUI / Vant Weapp | UI 组件库（选其一） |
| 微信云开发 | 云函数/云数据库/云存储（推荐，免运维） |
| 或自建后端 | Python FastAPI + MySQL |

### 2.2 后端（推荐微信云开发，以下为自建方案备选）
| 技术 | 说明 |
|------|------|
| Python FastAPI | RESTful API |
| SQLAlchemy 2.0 | ORM |
| PostgreSQL / MySQL | 关系型数据库 |
| Redis | 缓存/签到防刷 |
| Docker + Docker Compose | 容器化部署 |

### 2.3 推荐方案
**起步推荐**：微信云开发（云函数 + 云数据库 + 云存储），零运维成本。
**进阶方案**：自建 FastAPI 后端，灵活性更高。

## 三、数据模型设计

### 3.1 用户体系

```sql
-- 用户表
users (
  id          VARCHAR(64) PRIMARY KEY,      -- 微信 openid
  nickname    VARCHAR(64),                   -- 微信昵称
  avatar_url  VARCHAR(256),                  -- 头像 URL
  gender      TINYINT,                       -- 0:未知 1:男 2:女
  phone       VARCHAR(20),                   -- 手机号（可选）
  created_at  DATETIME DEFAULT NOW(),
  updated_at  DATETIME DEFAULT NOW()
);

-- 用户设置表
user_settings (
  id                  INT PRIMARY KEY AUTO_INCREMENT,
  user_id             VARCHAR(64) NOT NULL,
  daily_goal_step     INT DEFAULT 0,         -- 每日步数目标
  daily_goal_minutes  INT DEFAULT 30,        -- 每日运动时长目标（分钟）
  preferred_sports    TEXT,                   -- 偏好运动类型, JSON array
  remind_time         VARCHAR(20),            -- 提醒时间 e.g. 08:00
  created_at          DATETIME DEFAULT NOW(),
  updated_at          DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

### 3.2 健身运动 - FItness

```sql
-- 运动类型字典
sport_types (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  name        VARCHAR(32) NOT NULL,           -- 跑步/骑行/游泳/瑜伽/力量/...
  icon        VARCHAR(64),                    -- 图标 URL
  unit        VARCHAR(16),                    -- 单位: 分钟/公里/组
  calories_per_unit DECIMAL(6,2) DEFAULT 0,   -- 每单位消耗卡路里
  is_active   TINYINT DEFAULT 1,
  created_at  DATETIME DEFAULT NOW()
);

-- 运动记录
workout_records (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  user_id         VARCHAR(64) NOT NULL,
  sport_type_id   INT NOT NULL,
  duration        INT NOT NULL,               -- 时长（分钟）
  distance        DECIMAL(8,2) DEFAULT 0,     -- 距离（公里）
  calories        INT DEFAULT 0,              -- 消耗卡路里
  intensity       TINYINT DEFAULT 3,          -- 强度 1-5
  note            VARCHAR(256),               -- 备注
  sport_date      DATE NOT NULL,              -- 运动日期
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (sport_type_id) REFERENCES sport_types(id),
  INDEX idx_user_date (user_id, sport_date)
);

-- 运动打卡表（每日签到）
workout_checkins (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  user_id     VARCHAR(64) NOT NULL,
  checkin_date DATE NOT NULL,
  duration    INT DEFAULT 0,                  -- 当天累计运动时长
  is_goal_met TINYINT DEFAULT 0,              -- 是否完成目标
  streak_days INT DEFAULT 0,                  -- 连续打卡天数
  created_at  DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE KEY uk_user_date (user_id, checkin_date)
);
```

### 3.3 团队活动 - Team Activities

```sql
-- 团队/圈子
teams (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  name            VARCHAR(64) NOT NULL,
  description     TEXT,
  avatar_url      VARCHAR(256),
  owner_id        VARCHAR(64) NOT NULL,        -- 创建人
  member_count    INT DEFAULT 1,
  invite_code     VARCHAR(16) UNIQUE,          -- 邀请码
  status          TINYINT DEFAULT 1,           -- 1:活跃 0:解散
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (owner_id) REFERENCES users(id)
);

-- 团队成员
team_members (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  team_id     INT NOT NULL,
  user_id     VARCHAR(64) NOT NULL,
  role        TINYINT DEFAULT 0,               -- 0:成员 1:管理员 2:群主
  joined_at   DATETIME DEFAULT NOW(),
  FOREIGN KEY (team_id) REFERENCES teams(id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE KEY uk_team_user (team_id, user_id)
);

-- 团队活动
team_activities (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  team_id         INT NOT NULL,
  title           VARCHAR(128) NOT NULL,
  description     TEXT,
  location        VARCHAR(256),                -- 活动地点
  activity_date   DATETIME NOT NULL,           -- 活动时间
  deadline        DATETIME,                    -- 报名截止时间
  max_participants INT DEFAULT 0,              -- 0 表示不限制
  cover_image     VARCHAR(256),
  status          TINYINT DEFAULT 0,           -- 0:报名中 1:进行中 2:已结束 3:已取消
  created_by      VARCHAR(64) NOT NULL,
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (team_id) REFERENCES teams(id),
  FOREIGN KEY (created_by) REFERENCES users(id)
);

-- 活动报名
activity_participants (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  activity_id     INT NOT NULL,
  user_id         VARCHAR(64) NOT NULL,
  status          TINYINT DEFAULT 0,           -- 0:已报名 1:已签到 2:已取消
  checkin_time    DATETIME,                    -- 签到时间
  checkin_location POINT,                      -- 签到位置（可做位置限定）
  remark          VARCHAR(128),
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (activity_id) REFERENCES team_activities(id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE KEY uk_activity_user (activity_id, user_id)
);
```

### 3.4 个人习惯/任务 - Habits & Tasks

```sql
-- 习惯定义
habits (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  user_id         VARCHAR(64) NOT NULL,
  name            VARCHAR(64) NOT NULL,          -- e.g. "早起喝水"
  description     VARCHAR(256),
  frequency_type  TINYINT DEFAULT 0,             -- 0:每日 1:每周N次 2:自定义
  frequency_value VARCHAR(32),                   -- 根据 type: [1,3,5] / "mon,wed,fri"
  target_count    INT DEFAULT 1,                 -- 每日/次目标次数
  color           VARCHAR(7) DEFAULT '#4CAF50',  -- 颜色
  icon            VARCHAR(16) DEFAULT '✅',
  is_archived     TINYINT DEFAULT 0,
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 习惯打卡记录
habit_records (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  habit_id    INT NOT NULL,
  user_id     VARCHAR(64) NOT NULL,
  record_date DATE NOT NULL,
  count       INT DEFAULT 1,                     -- 完成次数
  note        VARCHAR(128),
  created_at  DATETIME DEFAULT NOW(),
  FOREIGN KEY (habit_id) REFERENCES habits(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE KEY uk_habit_date (habit_id, record_date)
);

-- 任务/待办
tasks (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  user_id         VARCHAR(64) NOT NULL,
  title           VARCHAR(128) NOT NULL,
  description     TEXT,
  priority        TINYINT DEFAULT 1,             -- 1:低 2:中 3:高
  due_date        DATE,                          -- 截止日期
  due_time        VARCHAR(10),                   -- e.g. "14:30"
  category        VARCHAR(32),                   -- 分类标签
  status          TINYINT DEFAULT 0,             -- 0:待办 1:进行中 2:已完成 3:已取消
  completed_at    DATETIME,
  sort_order      INT DEFAULT 0,                 -- 排序
  created_at      DATETIME DEFAULT NOW(),
  updated_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id),
  INDEX idx_user_status (user_id, status)
);

-- 任务复盘记录
task_reviews (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  user_id     VARCHAR(64) NOT NULL,
  review_date DATE NOT NULL,                     -- 复盘日期（可按日/周/月）
  period_type TINYINT DEFAULT 0,                 -- 0:日复盘 1:周复盘 2:月复盘
  completed_count INT DEFAULT 0,                 -- 完成任务数
  total_count     INT DEFAULT 0,                 -- 总任务数
  achievement     TEXT,                          -- 今日成就
  reflection      TEXT,                          -- 反思/不足
  tomorrow_plan   TEXT,                          -- 明日计划
  mood            TINYINT,                       -- 心情 1-5
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE KEY uk_user_period (user_id, review_date, period_type)
);
```

### 3.5 专项活动 - Special Events

```sql
-- 专项活动定义（可跑团、读书挑战、骑行赛等）
special_events (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  user_id         VARCHAR(64) NOT NULL,
  event_type      VARCHAR(32) NOT NULL,          -- running / reading / cycling / custom
  title           VARCHAR(128) NOT NULL,
  description     TEXT,
  start_date      DATE NOT NULL,
  end_date        DATE NOT NULL,
  goal_type       VARCHAR(16),                   -- distance / duration / count / pages
  goal_value      DECIMAL(10,2),                 -- 目标值
  current_value   DECIMAL(10,2) DEFAULT 0,       -- 当前进度
  is_team_event   TINYINT DEFAULT 0,             -- 是否团队挑战
  team_id         INT,                           -- 关联团队
  cover_image     VARCHAR(256),
  status          TINYINT DEFAULT 0,             -- 0:未开始 1:进行中 2:已完成 3:已放弃
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (user_id) REFERENCES users(id),
  FOREIGN KEY (team_id) REFERENCES teams(id)
);

-- 专项活动进度记录
special_event_logs (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  event_id        INT NOT NULL,
  user_id         VARCHAR(64) NOT NULL,
  log_date        DATE NOT NULL,
  value           DECIMAL(10,2) NOT NULL,        -- 本次贡献值
  note            VARCHAR(256),
  image_urls      TEXT,                          -- 图片证明, JSON array
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (event_id) REFERENCES special_events(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 专项排行榜（用于团队挑战排名）
special_event_rankings (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  event_id    INT NOT NULL,
  user_id     VARCHAR(64) NOT NULL,
  total_value DECIMAL(10,2) DEFAULT 0,
  rank_no     INT,
  updated_at  DATETIME DEFAULT NOW(),
  FOREIGN KEY (event_id) REFERENCES special_events(id) ON DELETE CASCADE,
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE KEY uk_event_user (event_id, user_id)
);
```

### 5.2 健身运动
| 方法 | 路径 | 说明 |
|------|------|------|
| GET  | /api/workouts | 获取运动记录列表 |

## 八、关键设计决策

### 8.1 打卡连续天数计算
- 使用 `workout_checkins.streak_days` 字段
- 每日首次打卡时查询前一天是否有打卡记录
- 有则 `streak_days = 前一天的 streak_days + 1`
- 无则 `streak_days = 1`
- 连续中断 >1 天则重置为 1

### 8.2 团队签到位置限制
- 可选：开启位置限制后，使用微信 `wx.getLocation`
- 对比活动地点经纬度，在指定半径内（如 500m）才能签到

### 8.3 复盘周期
- 日复盘：当晚自动生成模板，用户补充
- 周复盘：每周日生成过去 7 天汇总
- 月复盘：每月 1 号生成月度整体数据

### 8.4 专项活动排行榜
- 团队挑战时，按 `total_value` 降序排列
- 定期（每小时/每天）更新排名缓存
- 支持按队伍/个人维度查看
