# Tribo 重构方案 v2：团队活动协作工具

> 定位：面向**工作/项目团队**的活动跟踪与信息共享工具
> 核心理念：从「个人运动打卡」转型为「团队活动协作」

---

## 一、核心用户旅程

```
计划阶段                 执行阶段                   记录阶段
┌──────────┐    ┌──────────────────┐    ┌───────────────┐
│ 创建活动   │ → │ 成员报名 → 签到   │ → │ 活动总结/照片  │
│ 设定时间   │   │ 全员可见动态      │   │ 自动更新统计   │
│ 通知成员   │   │ 评论交流          │   │ 动态沉淀到团队  │
└──────────┘    └──────────────────┘    └───────────────┘
                              ↕
                    团队首页动态流（Feed）
                    所有操作实时可见
```

---

## 二、模块调整总表

| 模块 | 操作 | 原因 |
|------|:----:|------|
| **用户系统** | 🟢 保留，精简 | 登录/信息/认证保留；去掉运动相关设置 |
| **团队活动** | 🟢 **核心增强 🔥** | 增加 Feed/评论/公告/签到统计 |
| **健身运动** | 🔴 **删除** | 纯个人功能，与团队无关（工作团队不需要运动打卡） |
| **专项挑战** | 🟡 改造为「团队目标」 | 团队共同目标+进度追踪+排行榜，适合项目里程碑场景 |
| **成就系统** | 🟡 改造为团队成就 | 保留表结构，成就条件改为团队里程碑 |
| **任务/习惯** | 🔴 **删除** | 个人习惯/待办与团队无关 |
| **活动运营** | 🔴 **删除** | 审批流/礼品/赞助商/入场券全部冗余 |
| **数据统计** | 🟡 改造 | 从个人运动统计改为团队参与统计 |

## 三、变化详解

### 1. 删除模块（直接移除）

**健身运动** (`models/workout.py`, `routers/workout.py`, `schemas/workout.py`)
- 删除: WorkoutRecord, WorkoutCheckin, SportType
- 前端: 删除 workout/ 和 workout_log/ 页面
- 理由: 工作团队不需要跑步、骑行等运动打卡

**习惯/任务** (`models/habit.py`, `routers/habit.py`, `schemas/habit.py`)
- 删除: Habit, HabitRecord, Task, TaskReview
- 前端: 删除 tasks/ 和 habits/ 页面
- 理由: 个人习惯和待办与团队无关

**活动运营** (`models/ops.py`, `routers/ops.py`, `schemas/ops.py`)
- 删除: OpsActivity, OpsGift, OpsGiftRecord, OpsSponsor, OpsEnrollment, OpsReview
- 前端: 删除 ops_list/, ops_detail/, ops_enroll/, ops_ticket/ 页面
- 理由: 审批流/礼品/赞助商完全超出团队需求

### 2. 核心增强（重点投入 🔥）

**团队活动模块** — 从 4 张表扩展到 7+ 张表：

| 当前已有 | 保留 | 说明 |
|---------|:----:|------|
| Team | ✅ | 团队基本信息 |
| TeamMember | ✅ | 成员/角色/加入时间 |
| TeamActivity | ✅ **增强** | 增加活动总结字段、活动类型标签 |
| ActivityParticipant | ✅ **增强** | 增加签到时间、备注 |

| 需要新增 | 说明 |
|---------|------|
| TeamFeed | 团队动态流（谁做了什么事，自动生成） |
| ActivityComment | 活动下的讨论/图片 |
| TeamAnnouncement | 管理员发的团队公告 |
| Notification | 站内通知（新活动/提及/报名确认） |

**TeamActivity 增强字段**（适合工作团队）：
- `activity_type` — 会议/头脑风暴/团建/项目/培训/其他
- `summary` — 活动总结（活动后填写）
- `notes` — 活动纪要
- `photo_urls` — 活动照片 JSON

### 3. 改造模块

**专项挑战 → 团队目标** (TeamGoal)
- 定位：团队一起冲刺一个共同目标
  - "Q3 完成 100 个客户拜访"
  - "本月完成项目 MVP"
  - "团队背单词 3000 个"
- 保留：进度追踪、排行榜、进度环 UI
- 去掉：个人模式、运动类型
- 增加：关联团队

**成就系统 → 团队里程碑**
- 成就条件改造：
  - "首次举办活动"
  - "团队连续参与 5 次活动"
  - "成功完成 3 个团队目标"
  - "团队活跃度达到 80%"
- 保留：`Achievement` + `UserAchievement` 表结构

**数据统计 → 团队洞察**
- 团队活动总览（活动数、参与人次、签到率）
- 成员排行榜（个人参与次数/签到率）
- 团队活跃度趋势（周/月）
- 保留周/月趋势结构，换数据源

## 四、新增核心功能设计

### 1. 团队动态流 (Feed) — P0

**模型设计**:
```python
class TeamFeedItem(Base):
    __tablename__ = "team_feeds"
    
    id: int (PK)
    team_id: int
    actor_id: str       # 谁做的
    action_type: str    # create_activity / signup / checkin / join_team / add_comment
    target_type: str    # activity / team / comment
    target_id: int
    summary: str        # "张三报名了"Q3规划会议"
    created_at: datetime
```

- 所有关键操作自动写入 Feed
- GET `/api/teams/{id}/feed` 分页返回
- 前端首页 Tab 直接展示 Feed

### 2. 活动评论
```python
class ActivityComment(Base):
    __tablename__ = "activity_comments"
    
    id: int (PK)
    activity_id: int
    user_id: str
    content: str
    image_urls: str (JSON)  # 可选图片
    created_at: datetime
```

### 3. 通知
```python
class Notification(Base):
    __tablename__ = "notifications"
    
    id: int (PK)
    user_id: str
    type: str       # activity_remind / signup_confirmed / team_invite / new_comment
    title: str
    body: str
    ref_type: str   # activity / team
    ref_id: int
    is_read: int (default 0)
    created_at: datetime
```

## 七、关键设计决策

1. **活动全流程**: 一个 TeamActivity 覆盖计划→执行→记录三个阶段，不拆分多个模型
2. **Feed 自动生成**: 关键操作由后端自动写入 Feed，无需用户手动发布动态
3. **不引入即时通讯**: 用评论替代聊天，减少实时性要求，降低复杂度
4. **保留微信登录**: 用已有 JWT 认证，不改认证方案
5. **通知先做站内**: 后续再考虑微信订阅消息推送
