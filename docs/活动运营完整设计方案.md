# 🎯 Tribo 活动运营系统 · 完整设计方案

> 覆盖活动全生命周期：创建 → 审批 → 筹备 → 报名 → 现场 → 收尾

---

## 一、活动全生命周期流程

```
┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐
│ 创建活动 │ → │ 运营审批 │ → │ 筹备配置 │ → │ 报名参会 │ → │ 现场签到 │ → │ 活动收尾 │
│ (发起人) │   │ (管理员) │   │ (发起人) │   │ (会员)  │   │ (扫码)  │   │  (数据)  │
└─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘   └─────────┘
     │             │             │             │             │             │
     ├ 活动信息     ├ 审核通过    ├ 布景资料    ├ 会员信息    ├ 扫码签到    ├ 礼品发放
     │ 时间/地点   │ 驳回理由   │ 物料清单   │ 报名表单   │ 身份核验   │ 数据统计
     ├ 布景需求    └ 状态变更   ├ 场地布置   ├ 人数限制   ├ 签到记录   ├ 活动报告
     │ 物料清单                 │ 应急预案   │ 候补机制   │ 胸牌打印   └ 参与凭证
     ├ 赞助商                    └ 志愿者分配 └ 群聊建群   └ 导览地图
     └ 礼品登记
```

## 二、核心功能模块

### 模块 1：活动创建（小程序端发起）

**发起人**（任何用户）可以直接在小程序创建活动，填写以下信息：

```
▸ 基本信息
  ├ 活动标题 *          ─ 必填
  ├ 活动类型             ─ 运动/派对/讲座/户外/其他
  ├ 活动封面             ─ 上传图片
  ├ 活动简介             ─ 一句话描述
  ├ 活动详情             ─ 富文本/Markdown（含日程安排）
  └ 活动标签             ─ 多个标签，方便筛选

▸ 时间地点
  ├ 开始时间 *           ─ 日期+时间
  ├ 结束时间 *           ─ 日期+时间
  ├ 报名截止时间 *        ─ 日期+时间
  ├ 活动地点 *           ─ 文本 + 地图定位（小程序地图组件）
  └ 场地信息             ─ 场地名称、详细地址、交通指南

▸ 布景/物料需求          ← 关键功能
  ├ 布景描述             ─ 文字描述+参考图
  ├ 物料清单             ─ 表格：物品名/数量/规格/预算
  │   ├ 例：签到台 x1 
  │   ├ 例：背景板 x1（3m x 2m）
  │   ├ 例：音响设备 x1套
  │   └ 例：矿泉水 x50瓶
  ├ 是否由平台提供物料    ─ 是/否
  └ 备注需求             ─ 其他特殊要求

▸ 参与设置
  ├ 人数上限             ─ 0=不限制
  ├ 报名截止时间          ─ 自动计算或手动设置
  ├ 报名条件             ─ 无/仅会员/仅团队/积分门槛
  ├ 报名表单自定义        ─ 可配置额外字段
  │   ├ 姓名 *           ─ 默认
  │   ├ 手机号 *          ─ 默认
  │   └ 自定义字段        ─ 如：T恤尺码、饮食禁忌、紧急联系人...
  └ 是否允许取消报名       ─ 是/否

▸ 礼品登记               ← 关键功能
  ├ 礼品名称             ─ 如：纪念T恤、运动水壶
  ├ 礼品数量              ─ 总库存
  ├ 礼品图片              ─ 展示图
  ├ 发放方式              ─ 签到领取 / 结束后领取 / 抽奖
  ├ 每人限领              ─ 默认1
  └ 礼品规格              ─ 如：颜色/尺码（用户在报名时选择）

▸ 赞助商信息              ← 关键功能
  ├ 赞助商名称 *          
  ├ 赞助商 LOGO           ─ 上传图片
  ├ 赞助类型              ─ 冠名/金牌/银牌/实物/场地
  ├ 赞助内容描述           ─ 如：提供全部饮用水
  ├ 赞助商简介            ─ 展示文案
  ├ 赞助商链接            ─ 官网/公众号
  └ 展示优先级             ─ 排序权重
```

### 模块 2：运营审批

发起人提交活动后，需管理员审批：

```
▸ 审批列表（管理端）
  ├ 待审批 / 已通过 / 已驳回
  ├ 预览活动详情
  ├ 审批操作
  │   ├ 通过 ─ 可选填写审批意见
  │   └ 驳回 ─ 必填驳回原因
  └ 审批通知
      ├ 通过 → 推送至活动列表，通知发起人
      └ 驳回 → 通知发起人原因，允许修改后重新提交

▸ 审批规则
  ├ 自动审批（可选）        ─ 白名单用户自动通过
  ├ 内容安全审核            ─ 敏感词过滤
  └ 时间冲突检测            ─ 同一场地时间不可重复
```

### 模块 3：报名参会

```
▸ 活动详情页
  ├ 活动信息               ─ 全部活动详情展示
  ├ 赞助商展示              ─ LOGO + 简介
  ├ 礼品展示                ─ 图片 + 规格选择
  ├ 已报名人数              ─ 实时显示
  └ 我要报名按钮

▸ 报名流程
  1. 阅读活动详情
  2. 填写报名表单
     ├ 姓名 *（自动填充微信昵称）
     ├ 手机号 *（可手动输入）
     ├ 自定义字段（如 T恤尺码、饮食禁忌）
     └ 礼品规格选择（如有）
  3. 确认提交
  4. 生成电子票/二维码
     └ 我的 → 我的活动 → 查看入场券

▸ 报名管理
  ├ 取消报名（如在允许时间内）
  ├ 候补排队（满员后自动排位）
  └ 活动群聊（报名后自动入群）
```

### 模块 4：现场签到（扫码）

```
▸ 入场券
  ├ 二维码               ─ 动态生成，每人唯一
  ├ 活动信息               ─ 活动名称/时间/地点
  └ 个人信息               ─ 姓名/编号

▸ 签到方式
  ├ 主动扫码               ─ 参会者出示入场券二维码
  │                       活动方用小程序扫一扫核销
  └ 扫码枪模式             ─ 支持外接扫码枪（H5版）

▸ 签到逻辑
  ┌─────────────────────────────────┐
  │  扫码 → 查询活动 + 查询用户      │
  │     ├ 已签到 → 显示"重复签到"     │
  │     ├ 未报名 → 显示"未报名"       │
  │     ├ 活动未开始→ 显示"等待入场"  │
  │     └ 核验通过 → 标记签到         │
  │        ├ 记录签到时间             │
  │        ├ 记录签到设备             │
  │        ├ 发放礼品（如有）          │
  │        └ 签到成功动画 + 语音播报  │
  └─────────────────────────────────┘

▸ 签到统计（实时）
  ├ 已签到 / 已报名 / 未签到
  ├ 签到率
  ├ 签到时间分布
  └ 礼品发放进度
```

### 模块 5：活动后——数据与报告

```
▸ 活动报告（发起人/管理员可见）
  ├ 报名统计
  │   ├ 总报名数 / 签到数 / 签到率
  │   └ 报名趋势图
  ├ 会员画像
  │   ├ 性别比例 / 年龄分布
  │   └ 新老会员占比
  ├ 礼品发放记录
  │   ├ 发放总数 / 剩余
  │   └ 规格分布
  ├ 活动成本
  │   ├ 物料预算
  │   ├ 实际支出
  │   └ 赞助抵消
  └ 导出功能
      ├ 签到名单 CSV
      ├ 礼品发放清单 CSV
      └ 活动报告 PDF
```

## 三、数据模型设计

```sql
-- ==========================================
-- 活动主表
-- ==========================================
CREATE TABLE ops_activities (
  id                INT PRIMARY KEY AUTO_INCREMENT,
  -- 基本信息
  title             VARCHAR(128) NOT NULL,       -- 活动标题
  type              VARCHAR(32) DEFAULT 'other', -- 活动类型: sport/party/lecture/outdoor/other
  cover_image       VARCHAR(256) DEFAULT '',
  summary           VARCHAR(256) DEFAULT '',      -- 一句话简介
  detail            TEXT,                         -- 富文本详情(含日程)
  tags              TEXT DEFAULT '[]',            -- 标签 JSON array
  
  -- 时间地点
  start_time        DATETIME NOT NULL,
  end_time          DATETIME NOT NULL,
  enroll_deadline   DATETIME,                     -- 报名截止
  location_name     VARCHAR(256) NOT NULL,         -- 场地名称
  location_address  VARCHAR(512) DEFAULT '',       -- 详细地址
  location_lat      DECIMAL(10,6) DEFAULT 0,      -- 地图坐标
  location_lng      DECIMAL(10,6) DEFAULT 0,
  location_guide    TEXT,                          -- 交通指南
  
  -- 布景/物料
  scenery_desc      TEXT,                          -- 布景描述
  material_list     TEXT DEFAULT '[]',             -- 物料清单 JSON
  need_platform_material INT DEFAULT 0,            -- 需要平台提供物料
  
  -- 参与设置
  max_participants  INT DEFAULT 0,
  enroll_condition  VARCHAR(64) DEFAULT 'none',    -- 报名条件
  custom_fields     TEXT DEFAULT '[]',             -- 自定义表单字段 JSON
  allow_cancel      INT DEFAULT 1,                 -- 允许取消报名
  
  -- 审批
  status            INT DEFAULT 0,                 -- 0:草稿 1:待审批 2:已通过 3:已驳回 4:进行中 5:已结束 6:已取消
  review_comment    VARCHAR(256) DEFAULT '',        -- 审批意见
  reviewer_id       VARCHAR(64) DEFAULT '',
  reviewed_at       DATETIME,
  
  -- 归属
  creator_id        VARCHAR(64) NOT NULL,           -- 发起人
  created_at        DATETIME DEFAULT NOW(),
  updated_at        DATETIME DEFAULT NOW() ON UPDATE NOW()
);


-- ==========================================
-- 礼品表
-- ==========================================
CREATE TABLE ops_gifts (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  activity_id     INT NOT NULL,
  name            VARCHAR(64) NOT NULL,
  total_count     INT DEFAULT 0,
  image           VARCHAR(256) DEFAULT '',
  give_method     VARCHAR(32) DEFAULT 'checkin',  -- checkin / after / lottery
  limit_per_user  INT DEFAULT 1,
  specs           TEXT DEFAULT '[]',                -- 规格 JSON: [{name:"颜色",options:["红","蓝"]},...]
  sort_order      INT DEFAULT 0,
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (activity_id) REFERENCES ops_activities(id)
);


-- ==========================================
-- 礼品发放记录
-- ==========================================
CREATE TABLE ops_gift_records (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  gift_id         INT NOT NULL,
  activity_id     INT NOT NULL,
  user_id         VARCHAR(64) NOT NULL,
  spec_values     TEXT DEFAULT '{}',                -- 选择的规格
  given_at        DATETIME,                        -- 发放时间
  given_by        VARCHAR(64),                     -- 发放人
  status          INT DEFAULT 0,                   -- 0:待领取 1:已领取
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (gift_id) REFERENCES ops_gifts(id),
  FOREIGN KEY (activity_id) REFERENCES ops_activities(id)
);


-- ==========================================
-- 赞助商表
-- ==========================================
CREATE TABLE ops_sponsors (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  activity_id     INT NOT NULL,
  name            VARCHAR(128) NOT NULL,
  logo            VARCHAR(256) DEFAULT '',
  sponsor_type    VARCHAR(32) DEFAULT 'gold',      -- title/gold/silver/bronze/material/venue
  description     TEXT,
  content_detail  TEXT,                             -- 赞助内容详情
  link_url        VARCHAR(256) DEFAULT '',
  sort_order      INT DEFAULT 0,
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (activity_id) REFERENCES ops_activities(id)
);


-- ==========================================
-- 活动报名表
-- ==========================================
CREATE TABLE ops_enrollments (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  activity_id     INT NOT NULL,
  user_id         VARCHAR(64) NOT NULL,
  
  -- 报名信息
  name            VARCHAR(64) NOT NULL,
  phone           VARCHAR(20) NOT NULL,
  form_data       TEXT DEFAULT '{}',                -- 自定义字段 JSON
  
  -- 礼品选择
  gift_specs      TEXT DEFAULT '{}',                -- 礼品规格选择 JSON
  
  -- 状态
  status          INT DEFAULT 0,                    -- 0:已报名 1:已签到 2:已取消 3:候补
  
  -- 签到
  checkin_time    DATETIME,
  checkin_device  VARCHAR(64),
  
  -- 入场券
  ticket_code     VARCHAR(64) UNIQUE,              -- 唯一入场码/二维码
  
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (activity_id) REFERENCES ops_activities(id),
  FOREIGN KEY (user_id) REFERENCES users(id),
  UNIQUE KEY uk_activity_user (activity_id, user_id)
);


-- ==========================================
-- 审批记录表
-- ==========================================
CREATE TABLE ops_reviews (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  activity_id     INT NOT NULL,
  reviewer_id     VARCHAR(64) NOT NULL,
  action          VARCHAR(16) NOT NULL,            -- approved / rejected / returned
  comment         TEXT,
  created_at      DATETIME DEFAULT NOW(),
  FOREIGN KEY (activity_id) REFERENCES ops_activities(id)
);

```

## 四、API 设计

### 活动 CRUD
```
POST   /api/ops/activities              — 创建活动（用户端）
GET    /api/ops/activities              — 活动列表（筛选条件）
GET    /api/ops/activities/{id}         — 活动详情
PUT    /api/ops/activities/{id}         — 编辑活动
DELETE /api/ops/activities/{id}         — 删除活动（仅创建者或管理员）
```

### 审批
```
POST   /api/ops/activities/{id}/submit  — 提交审批
GET    /api/ops/reviews                 — 待审批列表（管理端）
POST   /api/ops/activities/{id}/review  — 审批操作（通过/驳回）
```

### 礼品
```
POST   /api/ops/activities/{id}/gifts          — 添加礼品
GET    /api/ops/activities/{id}/gifts           — 礼品列表
PUT    /api/ops/gifts/{id}                      — 编辑礼品
DELETE /api/ops/gifts/{id}                      — 删除礼品
GET    /api/ops/activities/{id}/gift-records    — 礼品发放记录
POST   /api/ops/give-gift                       — 发放礼品（扫码签到触发）
```

### 赞助商
```
POST   /api/ops/activities/{id}/sponsors        — 添加赞助商
GET    /api/ops/activities/{id}/sponsors         — 赞助商列表
PUT    /api/ops/sponsors/{id}                    — 编辑赞助商
DELETE /api/ops/sponsors/{id}                    — 删除赞助商
```

### 报名 & 签到
```
POST   /api/ops/activities/{id}/enroll          — 报名
POST   /api/ops/activities/{id}/cancel          — 取消报名
GET    /api/ops/activities/{id}/enrollments      — 报名列表（发起人/管理员）
GET    /api/ops/enrollments/{id}/ticket          — 入场券信息
POST   /api/ops/checkin                         — 扫码签到（设备扫入场券）
GET    /api/ops/activities/{id}/checkin-stats    — 签到统计
```

### 数据导出
```
GET    /api/ops/activities/{id}/export/enrollments  — 导出报名名单 CSV
GET    /api/ops/activities/{id}/export/gifts        — 导出礼品清单 CSV
GET    /api/ops/activities/{id}/report              — 活动报告
```

## 六、扫码签到流程详解

```
┌─────────────────────────────────────────┐
│           扫码签到全流程                    │
├─────────────────────────────────────────┤
│                                          │

│  │ 打开活动管理    │                       │

│         │ 扫参会者入场券二维码              │
│         ▼                                │
│  ┌──────────────────┐                    │
│  │ 解析 ticket_code  │                    │
│  │ → 查活动ID+用户ID │                    │

│         ▼                                │

│  │ 核验逻辑          │                    │

│  │ ❌ 已签到       → 显示"重复签到"        │

│         │                                │
│         ▼                                │

│  │ • 记录时间        │                    │
│  │ • 触发礼品发放    │                    │

│  └────────┬─────────┘                    │
│           │                              │
└───────────┼──────────────────────────────┘
            │
            ▼
┌──────────────────────┐
```
