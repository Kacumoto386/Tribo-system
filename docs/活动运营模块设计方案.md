# 📢 活动运营模块 — 设计方案

## 一、什么是"活动运营"？

活动运营 = **平台统一发起的活动管理**，区别于用户自建的团队活动和专项挑战：

| | 现有：团队活动 | 现有：专项挑战 | **新增：活动运营** |
|---|---|---|---|
| 发起方 | 团队管理员 | 用户自己 | **平台/运营方** |
| 范围 | 团队内部 | 个人 | **全平台用户** |
| 类型 | 线下聚会/训练 | 个人目标挑战 | **打卡赛/排行榜赛/积分活动/节日活动** |
| 奖励 | 无 | 成就感 | **积分/徽章/称号/实物** |
| 时效 | 单次活动 | 有起止日期 | **固定周期/赛季制** |
| 规则 | 自由 | 用户设目标 | **固定规则自动排名** |

## 三、活动类型

### 3.1 打卡挑战赛
固定周期内完成指定运动打卡次数

```
例：21天跑步打卡挑战
├─ 规则：每天至少跑步15分钟，连续21天
├─ 进度：用户手动打卡或自动同步运动记录
├─ 排行：按连续天数 / 打卡总数排序
└─ 奖励：完成得「🏅 21天跑者」徽章 + 200积分
```

### 3.2 排位赛
按运动成绩排名的竞技活动

```
例：本月跑量排行榜
├─ 规则：统计月度跑步总里程
├─ 排行：实时更新排名
├─ 奖励：前10名获得称号 + 积分
└─ 防作弊：需手动上传截图（运营审核）
```

### 3.3 限时任务
一次性完成指定目标

```
例：周末3公里竞速
├─ 规则：周六日跑一次3公里，上传用时
├─ 排行：按用时排名
├─ 奖励：前三名奖牌 + 积分
```

### 3.4 节日/主题活动
结合节日的特殊活动

```
例：国庆7天运动嘉年华
├─ 规则：7天内运动3次即可完成
├─ 进度：每日打卡，点亮7枚勋章
└─ 奖励：集齐7枚得国庆限定徽章
```

## 四、数据模型

### 4.1 活动定义表 (ops_activities)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | 主键 |
| title | string | 活动标题 |
| description | text | 活动简介 |
| rules | text | 活动规则(Markdown) |
| activity_type | enum | campaign / ranking / mission / holiday |
| cover_image | string | 封面图 |
| start_time | datetime | 开始时间 |
| end_time | datetime | 结束时间 |
| enroll_deadline | datetime | 报名截止 |
| min_participants | int | 最低参与人数 |
| max_participants | int | 上限(0不限) |
| status | int | 0:草稿 1:报名中 2:进行中 3:已结束 4:已取消 |
| reward_title | string | 完成称号(如'21天跑者') |
| reward_badge_code | string | 关联成就/徽章code |
| reward_points | int | 奖励积分 |
| created_by | string | 管理员id |
| created_at | datetime | 创建时间 |

## 五、API 设计

### 活动管理 (仅管理员)
```
POST   /api/ops/activities          — 创建活动
PUT    /api/ops/activities/{id}     — 编辑活动
DELETE /api/ops/activities/{id}     — 删除活动
GET    /api/ops/activities          — 管理列表(含统计)
POST   /api/ops/activities/{id}/publish   — 发布/下架
GET    /api/ops/activities/{id}/data      — 活动数据看板
```

### 活动参与 (用户端)
```
GET    /api/ops/activities          — 活动列表(进行中/即将开始)
GET    /api/ops/activities/{id}     — 活动详情
POST   /api/ops/activities/{id}/enroll    — 报名/取消报名
GET    /api/ops/activities/{id}/ranking   — 排行榜
GET    /api/ops/activities/{id}/my-progress — 个人进度
POST   /api/ops/activities/{id}/log       — 手动打卡(需审核)
```

### 积分系统
```
GET    /api/ops/points              — 我的积分
GET    /api/ops/points/history      — 积分明细
```
